## 7. ESCALATE conditions (when subagent must STOP and ask)

Subagent dispatched in Phase 2 MUST return with a `## ESCALATE` block (not silently proceed) if any of the following are encountered:

1. **Need to touch FROZEN engine source** (`Plugins/FrameSolver/Source/FrameCore/` or `Plugins/LevelSim/Source/LevelCore/`) — Iron rule #1/#2 amendment requires explicit user authorization.
2. **Need to modify ALS / SPUD / SUQS / Prefabricator plugin source** — anti-goal #3; defer unless explicitly authorized.
3. **Self-estimate budget overrun ≥ 2x** the planned hours, OR step cap reached at ≥ 150% threshold (e.g. unit budgeted 30 steps but reached 45+).
4. **Scenario UI requires voice synthesis SDK external dependency** that wasn't in plan (e.g. needs `pip install xxx-tts` or 3rd-party UE plugin) — user adjudicates dep choice + license.
5. **5-leg gate regression** — gate goes from PASS at start of unit to FAIL with `succeeded < expected` — fix locally first, but if root cause is outside unit scope (e.g. flaky test, infrastructure), ESCALATE.
6. **Naming-collision risk** in `FFrameModelDef` USTRUCT API surface or `UArchSimMemberData` UPROPERTY layout that would break SPUD save format compatibility with v0.3.x save files.
7. **PIE 5min smoke fails with a class of issue** that user must adjudicate (e.g. ALS state machine deadlock, Enhanced Input deadzone wrong, heatmap renders but values stale) — bring data + propose 2-3 fix paths, don't silently iterate.
