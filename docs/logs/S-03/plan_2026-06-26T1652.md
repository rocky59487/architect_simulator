## Risk-adjusted ordering rationale

Risk profile is **Experimental** but user split the session: **v0.3.0 path runs safe-first** (confidence-build LOW → MED → HIGH risk ordering), **v0.4.0 spike runs experimental-first** (highest-risk units first to fail-fast). Within v0.3.0:

- **LOW-batch first** — 5 small cleanups, no risk, builds reviewer baseline + green gate before touching anything load-bearing.
- **AS-17 second** — audit-style; may conclude no code change needed. Keeps confidence going.
- **AS-15 third, AS-16 fourth** — both HIGH refit, same-file co-tenancy → MUST sequential. AS-15 must run before AS-16 to prevent merge conflict on `ArchSimCharacter.{h,cpp}`.
- **AS-13-u1 fifth (highest v0.3.0 risk)** — PIE-world harness in `-nullrhi` is the bonk-risk unit. By placing it 5th, we have 4 green units already shipped if it bonks. Auto-fallback to `UWorld::CreateWorld` per HANDOFF_v0.2.0 §6. If BOTH bonk → ESCALATE, but units 1-4 still ship as v0.3.0.
- **AS-13-u2 sixth** — only meaningful if u1 green. Adds 3 deferred test branches.
- **RELEASE seventh** — bundles all into v0.3.0 tag (local, not pushed).

**Bottleneck:** AS-15 → AS-16 sequential is the rigid spine. AS-13-u1 → AS-13-u2 also rigid. Parallelism opportunity: Group A (LOW-batch) + Group B (AS-17) → could 2-spoke parallel for ~3h wall-clock save. Default plan dispatches sequentially for review-cost economy; user can request parallel if wall-clock pressure.

For **v0.4.0 spike** (eval-gated): UE5.8 spike FIRST because StructUtils deprecation result determines whether Scenario editor is even feasible on the new UE version. Scenario second. Auto-retreat to `Research/` on bonk, no main-tree pollution.
